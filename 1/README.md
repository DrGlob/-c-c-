# EEPROM/NOR по SPI — драйвер

Небольшой учебный пример: интерфейс SPI, bit‑banging через GPIO и два драйвера памяти — EEPROM 25LC040A и NOR W25Q128. Все сущности снабжены Doxygen‑комментариями, чтобы было удобно проверять и генерировать документацию.

**Что внутри**
- `eeprom.hpp` — все интерфейсы, bit‑banged SPI и оба драйвера.
- `test.cpp` — мини‑проверка логики записи EEPROM через mock‑SPI.

**Ключевые идеи**
- `ISpiBus`, `IGpioPin`, `IDelay` — минимальные абстракции “железа”.
- `BitBangSpiBus` — реализация SPI через вручную управляемые GPIO (SCK/MOSI/MISO).
- `SpiDevice` — обёртка с RAII‑управлением CS.
- `Eeprom25LC040A` — чтение/запись битов, байтов и массивов, запись постранично.
- `NorW25Q128` — отличие от EEPROM: отдельные erase‑команды, 24‑битные адреса, page‑program 256 байт, невозможность писать 0→1 без стирания.

**Как собрать тест**
```bash
clang++ -std=c++17 test.cpp -o t
./t
```
