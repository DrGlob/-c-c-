# Кастомный аллокатор (С)

Это решение учебного задания: реализовать собственные `malloc` и `free` для двух фиксированных размеров — 15 и 180 байт. В коде функции названы `my_malloc`/`my_free`, чтобы не конфликтовать со стандартной библиотекой (их можно переименовать в `malloc`/`free` при необходимости).

**Идея решения**
- Два пула фиксированного размера: под 15 и под 180 байт.
- Для каждого пула — свой список свободных блоков (free list).
- Память выдаётся из общего массива (`heap`) и режется на блоки нужного размера.
- Каждый блок содержит небольшой заголовок `Header` с классом размера, чтобы `my_free` мог вернуть блок в нужный пул.

**Добавленные защиты**
- Игнорирование `free` указателей не из `heap`.
- Проверка выравнивания указателя.
- Проверка “сдвинутых” указателей (адрес должен попадать в границу блока пула).
- Защита от двойного `free` через метку `CLS_FREE` в заголовке.

**Файлы**
- `allocator.h` — реализация `my_malloc` / `my_free`.
- `test.c` — простой тест.

**Как собрать и запустить**
```bash
clang test.c -o t
./t
```

**Ограничения**
- Поддерживаются только размеры 15 и 180 байт — все остальные запросы вернут `NULL`.
- Память берётся из статического массива (`heap`).
- Потокобезопасность не реализована (одно поточное использование).
